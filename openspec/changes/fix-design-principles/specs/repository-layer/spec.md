## ADDED Requirements

### Requirement: Repository class owns all SQL for its domain
Each domain lib (customers, contracts, assets, transactions, stores, tenants, users, reports, files, audit) SHALL have a `*.repository.ts` file that encapsulates all raw SQL queries. The corresponding `*.service.ts` SHALL NOT call `dataSource.query()` or `dataSource.transaction()` directly.

#### Scenario: Service delegates query to repository
- **WHEN** a service method needs to fetch data from the database
- **THEN** the service calls a typed method on the injected repository, not `dataSource.query()` directly

#### Scenario: Repository receives DataSource via injection
- **WHEN** the repository class is instantiated by the NestJS IoC container
- **THEN** it receives `DataSource` via constructor injection and stores it as a private readonly property

### Requirement: AuthService caches JWT keys at module init
`AuthService` SHALL read the private and public key files once during constructor initialization using `readFileSync`, storing them as `private readonly` instance properties. It SHALL NOT call `readFileSync` inside `login()`, `refresh()`, or any other per-request method.

#### Scenario: Keys loaded at bootstrap
- **WHEN** the NestJS application starts and `AuthModule` is initialized
- **THEN** `AuthService` constructor reads the key files exactly once

#### Scenario: Login uses cached key
- **WHEN** a user calls `POST /api/v1/auth/login`
- **THEN** the JWT is signed using the cached `this.privateKey` without any file system access

### Requirement: Contract code generation uses a single helper
`ContractsService` SHALL have exactly one private method responsible for generating the per-tenant/store/month contract code sequence. The `create()` method and any other callers SHALL invoke this single helper. Duplicate sequence-increment logic SHALL NOT exist outside this helper.

#### Scenario: No duplicate code paths
- **WHEN** a pawn contract is created
- **THEN** the contract code is generated by exactly one private helper method, not inline SQL
